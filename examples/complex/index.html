Then I started paying attention to "Web Components", which I'd heard about but dismissed without understanding.

Components are a set of WC3 specs for declarative scoped DOM.

html imports (instead of using js to fetch files)

<template> tags that aren't processed but can be instanced custom elements with their own private scope for CSS/JS/HTML

shadow dom - DOM that is private to an element, and can select 'public' children to sprinkle inside it's hidden structure as needed.

Polymer http://www.polymer-project.org/ is a framework for making components built on a polyfill shim to make it all work as it should until the specs are fully adopted.

I made a <class-toggle> element that toggles a class on itself or a target, and a <select-set> that has a value of its selected children. On the markup side it looks like:

<select-set use_class="active" multi=false> <class-toggle use_class="active"></class-toggle> <class-toggle use_class="active"></class-toggle> <class-toggle use_class="active"></class-toggle> </select-set>

Simple, declarative structure.

I could write a bit of javascript to get the same effect for a target HTML node, or construct something to insert, but in practice I don't reuse that kind of stuff and just reinvent the wheel every time.

Another element I made is <scrub-handle> which can be assigned an object.property to change based on a mouse axis (with a modifier and limits) You still have to assign the connections in js, but it's proving really useful any time I have some sort of drag / resize interaction.
